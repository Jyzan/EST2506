\section{系统概述与整体架构}

\subsection{系统简介}

本系统基于 S800 硬件板卡 TI TM4C1294NCPDT 作为下位机，Python 3.11 + PyQt5 作为 PC 上位机，通过 USB 虚拟串口实现 1:1 数字孪生镜像的双向协同。
四层架构分工如下：
\begin{itemize}
  \item 硬件外设层：S800 板卡 + I2C 7SEG 数码管 + I2C 按键 + GPIO USER1/USER2 + PCA9557 LED + 蜂鸣器，完成时钟走时、显示刷新、按键检测、闹钟触发等实时任务
  \item 串口协议层：UART0 中断驱动收发，实现命令容错识别、多参数组合提取及逆序响应
  \item PC 后台通信层：负责串口收发和 NTP、天气数据的异步请求
  \item PC UI 渲染层：完成数字孪生渲染、控制面板交互、四色日志显示和数据看板
\end{itemize}

\subsection{8 位 LED 辅助指示}
\begin{itemize}
    \item \textbf{LED0（系统心跳）}：1Hz 心跳。
    \item \textbf{LED1（闹钟状态）}：闹钟使能时常亮，正在响铃时闪烁。
    \item \textbf{LED2（编辑模式）}：日期、时间或闹钟编辑期间常亮。
    \item \textbf{LED3（UART 收发活动）}：每次有效串口收发后点亮约 100ms。
    \item \textbf{LED4（晴天状态）}：已收到天气且天气码为 SUN 时常亮。
    \item \textbf{LED5（雨雪状态）}：天气码为 RAI 或 SNO 时慢闪。
    \item \textbf{LED6（高温状态）}：温度不低于 30°C 时常亮。
    \item \textbf{LED7（NTP 同步状态）}：同步后常亮，超过 24 小时未再次同步时慢闪，未同步时熄灭。
\end{itemize}

这里的LED0–LED7分别对应S800板的LED1–LED8。此外，根据MCU与PC端开发要求 §15，当闹钟响铃且天气有效时，若为雨雪天气（RAI/SNO），额外延长 3 声蜂鸣；若高温（$\ge 30^\circ$C），8 位 LED 整体慢闪，仅在响铃期间生效，不影响其他模式的 LED 含义。

\section{协议规范与容错设计}

\subsection{通信参数}

\begin{itemize}
  \item 物理层：USB 虚拟串口
  \item 波特率：115200
  \item 数据格式：8 数据位，无校验，1 停止位，无流控
  \item 字符编码：ASCII
  \item 单帧最大长度：64 字节
\end{itemize}

\subsection{容错设计}

\textbf{大小写不敏感}：MCU收到命令后转大写，大小写命令如*SET:TIME 与 *set:time 完全等价。

\textbf{空格容错}：Process\_Command()按空格和 Tab 分割参数，任意数量的空格或 Tab 均可正确分割。

\textbf{缩写识别}：Token\_Matches()使用大写字母为必输前缀、小写字母可省略的策略。例如 MINute 最少须输入 3 字符 MIN，可写 MINU、MINUT等；MI 仅 2 字符拒绝；全大写词如ON必须完整输入。
\begin{lstlisting}[style=cstyle]
    char t[24], p[24];
    uint8_t req = 0, i;
    Upper_Copy(t, token, sizeof(t));
    Upper_Copy(p, pattern, sizeof(p));
    // 小写字母可以省略 其余字符必须输入
    for (i = 0; pattern[i]; i++) {
        if (!islower((unsigned char)pattern[i])) req++;
    }
    // token 长度必须 >= 必输字符数 <= 完整 pattern 长度
    if (strlen(t) < req || strlen(t) > strlen(p)) return false;
    return strncmp(p, t, strlen(t)) == 0;
}
\end{lstlisting}

\subsection{MCU 中断架构}

SysTick 1ms 时基驱动四个分频标志，Timer0A ISR 每 1ms 刷新一位数码管并分频采样按键，UART ISR 最高优先级处理字节接收。I2C 总线按 400kHz 配置；按键完成 20ms 防抖。运行期显示、LED 和按键 I2C 访问集中在 Timer0A ISR，异常恢复则由主循环暂停扫描后执行。UART 两级缓冲 ： g\_rx\_asm[ ] 行组装缓冲 → g\_rx\_ring[ ] 环形队列，行结束符到达时整行原子提交，主循环通过 UART\_GetLine() 取行调用 Process\_Command()。UART RX ISR 的整行提交逻辑（main.c:432-440）：

\section{扩展功能实现}

\subsection{E1 NTP 网络对时}

实现：ntp\_helper.py 三服务器容错（ntp\_helper.py:7-23），NetworkWorker QThread 异步执行不阻塞 UI。PC 点击按钮或 MCU 按 USER1 均走 ntp\_sync() 流程（main.py:901-935、main.py:1189-1224）。PC 按 *SET:DATE、*SET:TIME、\texttt{*NTP SYNC} 顺序逐条等待 OK（main.py:985-1018），任一步 ERROR 即中止，全部成功后才记录 SYNC。MCU LED7 在同步后常亮，超 24 小时未同步慢闪。MCU 端长按 USER1 键在数码管上短显 NTP 同步状态 \texttt{n.SY.xx} 共 3 秒：\texttt{n} 为距上次同步的小时数（0–9 钳位），\texttt{xx} 为 OK（24h 内）/ DR（漂移超 24h），若从未同步则显示 \texttt{\_.SY.NO}。

\subsection{E2 天气获取}

实现：weather\_helper.py 从 wttr.in 拉取并映射六种天气码（weather\_helper.py:5-45），通过 \texttt{*SET:WEA <T> <COND>} 下发到 MCU。LED4 晴常亮、LED5 雨雪慢闪、LED6 高温≥30°C 常亮（main.c:1825-1838）。USER2 短按短显天气 5 秒（main.c:1728-1730），期间 *EVT:DISP 正常上报 PC 镜像跟随。启动后 1 秒首次拉取，此后 30 分钟刷新；获取失败后每 60 秒自动重试。串口未连接时成功结果保存，连接后先补发缓存以恢复板上状态，再立即获取并下发最新天气。

\subsection{E3 自动昼夜模式}

实现：astral\_helper.py 按上海经纬度计算日出日落（astral\_helper.py:8-20），手动模式从 MCU 取当前时分代入判定（main.py:1271-1293）。DAY 正常时间显示完整 8 位并启用全部 LED；NIGHT 的空闲时间主界面仅显示时分 4 位（main.c:1259-1265），LED 仅保留心跳，基础闹钟与远程蜂鸣被抑制，日期、年份、编辑及临时显示不受限制。auto\_daynight() 每分钟检查，仅模式变化时下发。

\subsection{E4 数据可视化}

实现：依据MCU与PC端开发要求§28，log\_store.py 将 ALARM、SYNC、EDIT、KEY 四类数据持久化到 events.csv（log\_store.py:7-65）。CSV 文件包含 timestamp、type、data 三列，type 取值即上述四类事件，data 字段按事件类型各有不同格式（如 EDIT 为 \texttt{TYPE VALUE}、SYNC 为 \texttt{delta <ms>}），timestamp 为秒级 \texttt{YYYY-MM-DD HH:MM:SS} 不含毫秒。chart\_widget.py 用 matplotlib嵌入 Qt（chart\_widget.py:12-149），当前绘制闹钟触发时间、NTP 误差和按键热度三张图表，数据为空时显示 "No data"。

\section{自主增加功能：情景倒计时系统}

\subsection{动机}

传统闹钟难以覆盖专注学习、运动训练和临时任务等相对计时场景。本项目设计了软硬件协同的情景倒计时：MCU 可脱离 PC 独立编辑、运行和暂停，PC 则提供进度环、语音倒数与情景配置。其设计思路类似手机中的个性化铃声提醒，用户可根据任务性质选择醒目的滚动文本、具有辨识度的音乐旋律，或完全无声的视觉提示。音乐模式适合日常学习和训练，文本模式可直接表达任务内容；在课堂或夜间等不宜发声的环境中，静默模式仅通过数码管闪烁完成提醒，避免干扰周围人员。

\subsection{设计}

五态 FSM 为 IDLE → EDIT → RUN ↔ PAUSE → DONE → IDLE。EXT 短按进入编辑、启动、暂停或继续，长按停止（main.c:1516-1572、main.c:2063-2114）；板上编辑态由 SHIFT 切换分秒字段、ADD 加 1、SAVE 减 1，情景由 PC 下拉框或协议设置。三种完成情景为消息播报、音乐庆祝和静默提醒。协议扩展 *SET:COUNTDOWN TIME/SCENE/MSG/START/PAUSE/RESUME/STOP。

\begin{itemize}
  \item \texttt{*EVT:CD STATE <state> <remain> <total> <scene>}同步进度环与状态标签。
  \item \texttt{*EVT:CD DONE} 在归零瞬间单次上报，PC 据此触发完成状态与语音播报。
\end{itemize}

同时支持 \texttt{*GET:COUNTDOWN} 查询当前状态。PC 端 CountdownRing 圆弧进度环和 TTS 语音倒数 1-5 秒。MCU 复用显示、蜂鸣器和串口资源，PC 语音使用 requirements.txt 中声明的 pyttsx3 或 pywin32。

\subsection{实现}

MCU 状态和配置变量位于 main.c:145-183；1 秒递减、归零切换情景及事件上报位于 main.c:1758-1779；倒计时显示位于 Build\_Display() 的 main.c:1128-1178。音乐庆祝使用 PWM0 驱动 PK5，启动配置片段如下（main.c:1956-1964）：
\begin{lstlisting}[style=cstyle]
static void Music_Start(void)
{
    if (g_music_playing) return;
    // 首次调用时使能 PWM0 并配置引脚
    if (!g_pwm_configured) {
        SysCtlPeripheralEnable(SYSCTL_PERIPH_PWM0);

\end{lstlisting}

Music\_Service()按音符时长非阻塞步进并循环播放，Music\_Stop()关闭 PWM 后恢复 PK5 GPIO。情景 0 滚动自定义文本并使用独立节奏蜂鸣，不置位闹钟状态，因此 LED1 及其他原有 LED 指示保持原义；情景 1 闪烁 TIME UP 并持续播放音乐，由 FUNC、EXT 或 PC 控制区的“停止”按钮停止；情景 2 静默闪烁 TIME UP，10 秒后自动退出。Countdown\_HandleKey()处理板上编辑、暂停、继续和停止；Countdown\_HandleCommand()处理七种子命令。

PC 控制区位于 main.py:460-548，包含分秒、情景、文本、开始、暂停、停止和语音控件；CountdownRing（twin\_panel.py:289-352）按 remain/total 绘制进度；协议解析位于 protocol.py:40-68；完成与语音联动位于 main.py:1069-1117，TtsSpeaker 定义于 main.py:67-168。

\subsection{演示}

\begin{enumerate}
  \item 在UI界面或板子上点击EXT开始编辑，设置时长，如10 秒，再次点击开始
  \item 数码管显示 Cd0010，PC 进度环按剩余比例同步收缩
  \item 最后 5 秒语音倒数（可选），数码管与 PC 进度环每秒递减
  \item 到点情景0流水显示文本；情景1闪烁TIME UP并播放音乐；情景2静默闪烁
  \item 运行中可 EXT 暂停/继续，长按 EXT 停止
\end{enumerate}

\section{难点解决}

\subsection{心跳刷屏}

1Hz 的 *PONG、*EVT:DISP、*EVT:LED 每秒追加 3 行日志，运行数分钟导致卡顿。

解决：main.py中使用 setMaximumBlockCount(1000) 限制最大行数；protocol.py中使用is\_heartbeat() 过滤高频心跳包；「显示心跳」复选框默认关闭，过滤后日志区主要显示闹钟、按键、ERROR 等。

\subsection{串口断连与自动重连}

拔插 USB 或心跳超时后，旧 SerialWorker 可能仍有 queued signal 尚未送达；若直接创建新线程，可能出现旧信号误更新 UI 或重复处理协议行。

解决：心跳 3s 超时或串口读写直接异常后进入自动重连状态（main.py:690-797、main.py:1332-1345），关闭旧 worker 并在 1.5s 后重新扫描端口；创建新 worker 前统一 disconnect 旧信号并 deleteLater；回调入口检查 sender 是否为当前 worker，晚到旧信号直接丢弃。手动断开不会触发自动重连。

\subsection{显示与蜂鸣资源抢占}

时钟、消息、天气、倒计时、闹钟都会争用 8 位数码管和蜂鸣器。若简单覆盖状态，容易出现倒计时结束后不恢复时钟、远程 BEEP 打断闹钟、夜间模式仍响铃等问题。

解决：Build\_Display() 固定优先级为倒计时 > 天气短显 > 消息 > 编辑与 DISP 显示 > 时钟；BEEP 使用独立计时状态，不覆盖闹钟 ringing；夜间模式抑制基础闹钟和远程蜂鸣，保持显示和状态上报。

\section{补充说明}
\begin{itemize}
  \item \textbf{数字孪生镜像面板按键}: 《MCU与PC端开发要求》中第23条要求图片中只出现了8个按键FUNC、SHIFT、ADD、SAVE、USR1、USR2、K7、K8，但在《大作业题目》4.1 P3中明确要求实现十位按键，因此UI界面增加了DISP、SPEED两个按键。上方为 8 个 I2C 虚拟按键，包含 FUNC/SHIFT/ADD/
     SAVE/DISP/SPEED/FORMAT/EXT；下方为独立 USR1/USR2，实现视觉区分。
  \item \textbf{昼夜模式时基}: 自动昼夜的定时检查用板子上的时间; 手动「应用昼夜」按钮先向 MCU 查询当前钟表时间(*GET:TIME 返回 HH.MM.SS), 再由 current\_daynight 代入计算。
  \item \textbf{*GET:MODE 查询}：《大作业题目》5.2 的 GET 子命令仅列 DATE、TIME、ALARM、DISPLAY、FORMAT 五项。为支持 PC 连接后即时获知当前昼夜模式和状态栏实时更新，MCU 额外支持 \texttt{*GET:MODE} 查询，应答 \texttt{OK DAY} 或 \texttt{OK NIGHT}，并遵循 FORMAT RIGHT 逆序规则。
  \item \textbf{ERROR LEN 提示}：《MCU与PC端开发要求》中要求行长超过 64 后提示 ERROR LINE TOO LONG，但《大作业题目》5.2要求ERROR只能为SYNTAX/PARAM/RANGE/LEN/BUSY其中之一，故实现上采用ERROR LEN的提示信息。
  \item \textbf{容错规则验证}: 协议演示页提供缩写演示按钮(如 DISP 代替 DISPlay)和大小写混合演示按钮(如 *sEt:FoRmAt rIgHt), 可在不连接 MCU 的情况下直观理解容错的运作机制。
  \item \textbf{SAVE按键功能}：在自主情景倒计时扩展功能触发时，为了方便设置相对时间，引入了SAVE按键作为控制减一的编辑键，当倒计时结束之后，SAVE按键恢复成和文档要求一样的功能。
  \item \textbf{NIGHT模式显示状态}：文档要求NIGHT模式只显示时分四位，同时要求编辑状态应该完整显示，因此实现上NIGHT的四位限制只作用于空闲状态下的正常时间主界面。日期、年份、编辑、天气、消息和倒计时仍完整显示。
  \item \textbf{SET:KEY USER1说明}：物理 USER1 短按通过 *EVT:KEY USER1 请求 PC 对时，长按显示同步状态。PC 虚拟 USER1 短按直接启动同一 NTP 对时流程，长按下发 *SET:KEY USER1 触发板上同步状态短显，从而区分两种行为，否则在UI界面长按USER1后S800将不会显示n.SY.xx。
  \item \textbf{PC状态补查机制}：部分\texttt{SET}命令和\texttt{RST}命令成功后 MCU 只返回 \texttt{OK}，为避免 UI 状态栏未同步更新，PC 在下发部分\texttt{SET}指令或者\texttt{RST}指令后会补发\texttt{GET}查询命令。
\end{itemize}
