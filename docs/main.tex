\section{系统概述与整体架构}

\subsection{系统简介}

本系统基于 S800硬件板卡 TI TM4C1294 作为下位机，Python 3.11 + PyQt5 作为 PC 上位机，通过 USB 虚拟串口实现 1:1 数字孪生镜像的双向协同。

四层架构分工：
\begin{itemize}
  \item 硬件外设层：S800 板卡 + I2C 7SEG 数码管 + I2C 按键 + GPIO USER1/USER2 + PCA9557 LED + 蜂鸣器，完成时钟走时、显示刷新、按键检测、闹钟触发等实时任务
  \item 串口协议层：UART0 中断驱动收发 + ring buffer 行拼装 + Token\_Matches() 容错解析，实现命令容错识别、多参数组合提取及 FORMAT RIGHT 逆序响应
  \item PC 后台通信层：SerialWorker QThread + NetworkWorker QThread，负责串口收发和 NTP、天气数据的异步请求
  \item PC UI 渲染层：PyQt5 MainWindow + TwinPanel + QPlainTextEdit 日志，完成数字孪生渲染、控制面板交互、四色日志显示和数据看板
\end{itemize}

\subsection{系统架构}

数据流四主线：MCU 物理按键 → *EVT:KEY → PC 镜像面板脉冲高亮；PC 虚拟按键 → *SET:KEY → MCU Handle\_Key() 执行动作；PC 控制面板 → *SET:DATE/TIME... → MCU Process\_Command() 接收；PC 1Hz PING → *PONG 应答 → 延迟在线状态维护。PC 端 handle\_protocol\_line() 串联四条主线，统一分发到 twin.update\_digits/leds()、add\_log() 四色着色、update\_status() 状态栏刷新。

\subsection{通信参数}

\begin{itemize}

  \item 波特率：115200
  \item 数据格式：8 数据位，无校验，1 停止位，无流控

  \item 行结束符：CR LF 或 LF 兼容
  \item 单帧最大长度：64 字节

\subsection{容错三件套}

\textbf{大小写不敏感}：MCU Process\_Command() 收到一行后先调用 Upper\_Copy()（main.c:2042）全转大写，故 *SET:TIME 与 *set:time 完全等价。

\textbf{空格容错}：按空格和 Tab 分割参数（main.c:2127），任意数量的空格或 Tab 均可正确分割。

\textbf{缩写识别}：Token\_Matches()（main.c:2053）用模式串中大写字母为必输前缀、小写字母可省略的策略。例如 MINute 最少须输入 3 字符 MIN，可写 MINU、MINUT、MINUTE；MI 仅 2 字符拒绝；全大写词如 ON、OFF 必须完整输入。

Upper\_Copy 与 Token\_Matches 实现（main.c:2042-2061）：
\begin{lstlisting}[style=cstyle]
static void Upper_Copy(char *dst, const char *src, uint16_t max) {
    uint16_t i;
    for (i = 0; i < max - 1 && src[i]; i++)
        dst[i] = (char)toupper((unsigned char)src[i]);
    dst[i] = '\0';
}

static bool Token_Matches(const char *token, const char *pattern) {
    // token 长度必须在[必输字符数, 完整pattern长度]之间且为前缀
    uint8_t req = 0, i;
    for (i = 0; pattern[i]; i++)
        if (isupper((unsigned char)pattern[i])) req++;
    if (strlen(token) < req || strlen(token) > strlen(pattern)) return false;
    return strncmp(pattern_upper, token_upper, strlen(token)) == 0;
}
\end{lstlisting}

\subsection{FORMAT RIGHT 逆序}

Send\_OK\_Value()（main.c:2097）在 g\_format 为 FMT\_RIGHT 时逐字符逆序后再发送 OK 应答：当前时间 12:30:45，LEFT 应答 OK 12.30.45，RIGHT 应答 OK 54.03.21。PC 端 parse\_disp\_event()（protocol.py:12）和 handle\_ok()（main.py:848）负责解析逆序报文并还原。

\section{关键代码片段与实现}

\subsection{MCU 中断架构}

SysTick 1ms 时基驱动四个分频标志（main.c:273），Timer0A ISR 每 1ms 刷新一位数码管并分频采样按键（main.c:290），UART ISR 最高优先级处理字节接收（main.c:318）。所有 I2C 访问集中在 Timer0A ISR 内消除总线竞争，LED 用 g\_led\_dirty 标志延迟写入避免阻塞。

UART 两级缓冲 ： g\_rx\_asm[ ] 行组装缓冲 → g\_rx\_ring[ ] 环形队列，行结束符到达时整行原子提交，主循环通过 UART\_GetLine() 取行调用 Process\_Command()。

UART RX ISR 行组装逻辑（main.c:352）：
\begin{lstlisting}[style=cstyle]
static void UART_RxIsrHandler(uint8_t byte) {
    if (byte == '\r' || byte == '\n') {
        if (g_rx_asm_len == 0) return;  // 忽略空行
        Mark_Serial_Activity();
        // 整行原子提交到环形缓冲并递增行计数
        for (i = 0; i < g_rx_asm_len; i++)
            g_rx_ring[g_rx_head++] = g_rx_asm[i];
        g_rx_ring[g_rx_head++] = '\n';
        rx_line_ready++;
        g_rx_asm_len = 0; return;
    }
    if (g_rx_asm_len >= LINE_MAX) {
        g_rx_asm_ovf = true; return;  // 超长行丢弃 设标志后续回传 ERROR LEN
    }
    g_rx_asm[g_rx_asm_len++] = (char)byte;
}
\end{lstlisting}

\subsection{数字孪生面板}

\textbf{7SEG 自绘}：SevenSegmentDigit paintEvent（twin\_panel.py:105）中每个段用 QPainter drawRoundedRect，亮色 \#FF3030、暗色 \#220000、背景 \#050505，38 字符映射表与 MCU 段码完全一致。

def handle_protocol_line(self, line: str):

\section{自主增加功能：情景倒计时系统}

\subsection{动机}

传统闹钟基于绝对时刻触发，不适应演示、专注训练、短时提醒等需要相对时长的场景。本功能与基础闹钟互补：闹钟按绝对时间、倒计时按相对时长，二者独立并行。创新在于板上按键可独立编辑运行、三情景收尾、PC 进度环 + 语音联动。

\subsection{设计}

五态 FSM：IDLE → EDIT → RUN ↔ PAUSE → DONE → IDLE。EXT 键短按进入编辑/启动/暂停/继续，长按停止。编辑态 SHIFT 分秒字段切换、ADD 加 1、SAVE 减 1。三种情景由 g\_cd\_scene 选择：0 滚动文本（到点流水 g\_cd\_msg）、1 闪烁庆祝（DONE 全屏快闪）、2 静默（仅显 --00--）。

协议扩展 *SET:COUNTDOWN TIME/SCENE/MSG/START/PAUSE/RESUME/STOP。

\begin{itemize}
  \item \texttt{*EVT:CD STATE <state> <remain> <total> <scene>} 1Hz 心跳上报，PC 据此同步进度环与状态标签。state 取值 IDLE / EDIT / RUN / PAUSE / DONE，remain 和 total 为剩余秒与总秒，scene 取值 0 滚动文本、1 闪烁庆祝、2 静默。
  \item \texttt{*EVT:CD DONE} 归零瞬间单次上报，PC 据此触发语音播报到点文本并停定时器。
\end{itemize}

同时支持 \texttt{*GET:COUNTDOWN} 查询当前倒计时状态。PC 端 CountdownRing 圆弧进度环 + TTS 语音倒数 1-5 秒。RUN/PAUSE 态 8 LED 按剩余比例进度条，DONE 态全闪。完全复用 MCU 既有蜂鸣器、显示、LED 和串口机制，不新增依赖。

\subsection{实现}

MCU 状态枚举与全局变量定义了 CD\_IDLE/EDIT/RUN/PAUSE/DONE 五态，g\_cd\_total\_s 1-3599 可设定时长，g\_cd\_scene 0/1/2 选情景，g\_cd\_msg 存滚动文本。

核心按键处理 Countdown\_HandleKey()（main.c:1810）：
\begin{lstlisting}[style=cstyle]
static bool Countdown_HandleKey(key_code_t key) {
    if (g_cd_state == CD_IDLE) {
        if (key == KEY_EXT && g_edit_state == ST_IDLE) {
            g_cd_state = CD_EDIT; g_cd_field = 0; Send_CD_Event();
            return true;
        }}
    switch (g_cd_state) {
    case CD_EDIT:
        if (key == KEY_SHIFT) g_cd_field = (g_cd_field+1)%2; // 分秒切换
        if (key == KEY_ADD)   /* 当前字段+1 */;
        if (key == KEY_SAVE)  /* 当前字段-1 */;
        if (key == KEY_EXT)   Countdown_Start(); // 确认启动
    case CD_RUN:  if (key == KEY_EXT) {PAUSE; Send_CD_Event();}
    case CD_PAUSE: if (key == KEY_EXT) {RUN;   Send_CD_Event();}
    case CD_DONE:  if (key == KEY_EXT||key == KEY_FUNC) Countdown_Stop();
    }
}
\end{lstlisting}

Build\_Display() 中倒计时优先级最高：EDIT 显示 CdMMSS 闪烁高亮字段，RUN/PAUSE 显示 Cd 加剩余秒数，DONE 按情景设置滚动文本/全 DONE 闪烁/--00--，均设 10 秒超时兜底。Time\_Tick\_1s() 中 RUN 态递减 g\_cd\_remain\_s，归零转 DONE 并上报 *EVT:CD DONE。

协议命令族 Countdown\_HandleCommand()（main.c:1866）支持 TIME/SCENE/MSG/START/PAUSE/RESUME/STOP 七种子命令，以及向后兼容的 *SET:COUNTDOWN 数字直设时长。

PC 端 CountdownRing（twin\_panel.py:258）为 QWidget 自绘，drawArc 按 remain/total 比例渲染圆弧，中心显示剩余分秒和状态标签。TtsSpeaker（main.py:65）封装 pyttsx3 和 win32com SAPI 双通道，倒计时最后 5 秒语音播报，DONE 播报到点文本。UI 提供分秒 SpinBox、情景下拉框、文本输入、开始/暂停/停止按钮和语音复选框。

\subsection{使用说明}

\begin{enumerate}
  \item 在UI界面或板子上点击EXT开始编辑，设置时长，如10 秒，再次点击开始
  \item 数码管显示 Cd0010，8 LED 进度条按剩余比例逐位点亮，PC 进度环同步收缩
  \item 最后 5 秒语音倒数（可选），每秒递减，LED 逐位熄灭
  \item 归零后：情景 0 流水文本 + 蜂鸣器节奏 10 秒；情景 1 全屏 DONE 闪烁 + 蜂鸣快闪；情景 2 静显--00-- ，10 秒无操作自动退出
  \item 运行中可 EXT 暂停/继续，长按 EXT 停止；闹钟功能独立可用
\end{enumerate}

\section{难点解决与防崩溃机制}

\subsection{心跳刷屏}

1Hz 的 *PONG、*EVT:DISP、*EVT:LED 每秒追加 3 行日志，运行数分钟导致卡顿。

解决：setMaximumBlockCount(1000) 限制最大行数（main.py:606）；is\_heartbeat() 过滤心跳包（protocol.py:8）；「显示心跳」复选框（默认关闭）过滤后日志区仅显示闹钟、按键、ERROR 等事件。

\subsection{网络超时 UI 假死}

NTP/天气网络请求阻塞 5-10 秒，Qt 事件循环被卡住无法操作。

解决：NetworkWorker(QThread)（main.py:46）子线程执行，done/failed 信号跨线程回调，GUI 全程流畅，三 NTP 服务器自动切换容错。

\section{补充说明}
\begin{itemize}
  \item \textbf{数字孪生镜像面板按键布局}: 上方 8 个 I\textsuperscript{2}C 按键(FUNC / SHFT / ADD / SAVE / DISP / SPEED / FORMAT / EXT)以 QGridLayout 2行×4列排列; 中间以 QFrame.HLine 分割线隔开; 下方 2 个 GPIO 按键(USR1 / USR2)以蓝色调独立样式水平排列, 实现与 I\textsuperscript{2}C 8 键的视觉区分。

  \item \textbf{SAVE按键功能}：在自主情景倒计时扩展功能触发时，为了方便设置相对时间，引入了SAVE按键作为控制减一的编辑键，当倒计时结束之后，SAVE按键恢复成和文档要求一样的功能。
\end{itemize}
